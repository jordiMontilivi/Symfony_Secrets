---

### **ChatGPT**

JWT_PASSPHRASE és una clau secreta (una “contrasenya llarga”) que s’utilitza per signar i verificar els tokens JWT (JSON Web Tokens). Serveix perquè el servidor pugui comprovar que un token no ha estat manipulat.
🔐 Què fa exactament?

Quan el servidor crea un JWT, el signa amb la JWT_PASSPHRASE.

Quan rep el token, el torna a verificar amb aquesta mateixa clau.

Si la clau no coincideix → el token és invàlid.

👩‍🎓 Els alumnes tenen aquesta clau?
👉 No.
La JWT_PASSPHRASE mai s’ha de compartir amb els usuaris (alumnes).

És privada del servidor/backend.

Si un alumne la tingués, podria crear tokens vàlids falsos (greu problema de seguretat).

⚙️ Qui la crea?
👉 La crea el desenvolupador o l’administrador del sistema, no els alumnes.

Normalment es defineix en un fitxer .env:

```

```

Ha de ser:

Llarga (mínim 32+ caràcters)

Difícil d’endevinar

No compartida

❓ Els alumnes poden crear-la “al moment”?
👉 No té sentit en un sistema real.

Els JWT s’han de validar amb una única clau centralitzada.

Si cada usuari tingués la seva:

El servidor no podria verificar correctament els tokens.

Es trencaria el model de seguretat.

🧠 Resum ràpid

JWT_PASSPHRASE = clau secreta del servidor

❌ No la tenen els alumnes

❌ No la creen els usuaris
